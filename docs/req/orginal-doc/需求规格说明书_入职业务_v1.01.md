## 5.7 合同模块

本模块定义合同自动生成方案，基于入职员工的维度信息，自动确定应签合同清单、匹配合同模板、填充文本字段、预览确认，最终通过电子签或纸质签完成合同签署与归档。

合同/协议/承诺书共 13 种，覆盖 2 个基地 × 6 种用工形式；管理维度 9 项，字段约 50 个（去重）；签订规则分两类：A签（员工签字 + 公司用印，7 种）和 B签（仅员工签字，6 种）。

### 5.7.2 确定应签合同清单

**试用期上限校验**（依据《劳动合同法》第十九条）：

### 5.7.6 合同签署与归档

合同确认后，根据签署方式进入签署流程。支持线上电子签和线下纸质签双通道。

**签订规则分类**：

\- A签（员工签字 + 公司用印，一式两份，具备双边法律效力）：劳动合同、劳务协议、实习协议、非全日制劳动合同、补充协议（竞业）、保密协议、廉洁协议

\- B签（仅员工签字，一份即可，单方承诺性质）：网络与数据安全责任承诺书、个人健康状况承诺书、入职承诺书、无招聘费用返费承诺书、无竞业限制申明、职业病危害因素告知书

签署方式与流程：

| **签署方式** | **流程** | **说明** |
| --- | --- | --- |
| 线上签署（默认） | 合同确认 → 推送至 E签宝平台 → 员工移动端电子签署（飞书通知推送） → A签：公司电子章用印 / B签：跳过用印 → 签署完成 → 自动回传签署凭证 → 归档 | 集成 E签宝 |
| 线下签署（备选） | 合同确认 → 打印纸质合同 → 员工现场签字 → A签：发起 OA 用印申请 → 用印完成 / B签：跳过用印 → 扫描上传 → 归档 | 员工无法使用电子签时 |

**签署方式选择规则**：

| **场景** | **推荐方式** | **说明** |
| --- | --- | --- |
| 常规入职 | 线上（电子签） | 默认方式 |

### 5.7.7 合同生成引擎设计

**模板匹配流程（两步匹配）**：

1\. **维度匹配**：根据入职工单中的维度值（管控单位、用工单位、用工形式等），在维度映射表中找到匹配的映射记录，获取模板 ID

2\. **模板验证**：根据模板 ID 查询模板记录，验证 生效日期 ≤ 当前日期 < 失效日期，确认模板当前可用

**维度匹配优先级（当多个维度映射记录匹配到相同维度时）**：

1\. **完全匹配**：所有维度值完全一致 → 优先使用

2\. **通配匹配**：存在通配维度（如"\*"） → 降级使用

3\. **默认模板**：无任何匹配 → 使用默认模板

异常处理：

| **异常场景** | **处理方式** |
| --- | --- |
| 用工单位未配置法人信息 | "阻断生成，提示""该用工单位尚未维护法人信息，请先配置""" |
| 维度映射无匹配记录 | "阻断生成，提示""该维度组合未配置模板映射，请检查维度映射表""" |
| 映射的模板已失效 | "阻断生成，提示""映射的模板已失效（失效日期：{日期}），请更新维度映射""" |
| 必填字段缺失 | 阻断生成，提示补录 |
| 部分场景字段缺失 | 预览时发现为空，自行补录 |
| 职业病告知书已选择但危害因素未填 | 生成告知书，相关字段留空，提示合同经办人补充填入 |
| 试用期超法定上限 | 阻断生成，提示 HR 修正试用期或合同期限 |
| E签宝推送失败 | 自动重试（最多 3 次，间隔 30 秒）；仍失败提示同步电子签失败，合同经办人可手工推送 |
| E签宝签署回调超时（>24h） | 系统发送催办通知至员工和入职经办人；>72h 升级通知合同经办人 |

合同编号生成规则：

编号格式：**{管控单位简称}-{合同类型代码}-{年份}-{流水号}**

示例：SLYS-LDHT-2026-000001

| 组成部分 | 说明 | 示例 |
| --- | --- | --- |
| 管控单位简称 | 管控单位缩写，2-4 位大写字母 | JTZB（集团总部） |
| 合同类型代码 | 参见合同类型代码表 | LDHT（劳动合同） |
| 年份 | 合同生成当年的四位年份 | 2026 |
| 流水号 | 6 位数字，按「管控单位 + 合同类型 + 年份」分段递增 | 000001 |

\- 流水号按 管控单位 + 合同类型 + 年份 组合独立递增，全年内不归零

\- 编号在合同文本填充时生成，不占用空号

\- 合同作废后编号不回收、不重用

### 5.7.8 合同模板管理

合同模板采用模板表与维度映射表分离的设计：合同模板表管理模板文件本身（上传、版本、生效/失效日期），维度映射表管理维度组合与模板 ID 的映射关系。同一模板可被多个维度组合复用。

模板生效判断机制：

模板没有"有效/无效"等状态字段，系统通过生效日期和失效日期动态判断：

| 时间阶段 | 日期条件 | 是否参与合同生成 | 说明 |
| --- | --- | --- | --- |
| 未生效 | 当前日期 < 生效日期 | ❌ | 等待生效日期到达 |
| 生效中 | 生效日期 ≤ 当前日期 < 失效日期 | ✓ | 系统匹配模板时自动筛选 |
| 已失效 | 当前日期 ≥ 失效日期 | ❌ | 保留在系统中，供历史合同回溯查询 |

\- 上传新模板时：失效日期默认为 9999-12-31，表示永久生效

\- 上传新版本时：系统自动将同名旧模板的失效日期设为新模板生效日期 - 1 天

\- 失效模板保留在系统中，供历史合同回溯查询

模板新增/上传流程：

| 步骤 | 操作 | 说明 |
| --- | --- | --- |
| 1 | 选择合同类型 | 从 13 种合同类型 + 2 种辅助模板类型中选择 |
| 2 | 上传模板文件 | 上传 .docx 格式文件 |
| 3 | 填写模板名称 | 同一合同类型下名称唯一 |
| 4 | 设定生效日期 | 选择该模板开始生效的日期 |
| 5 | 提交保存 | 系统自动校验后保存，失效日期默认 9999-12-31 |

| **需求名称** | **详细要求** |

## 7.4 故障处理

| **故障描述** | **详细要求** |
| --- | --- |
| **服务器故障** | 应用服务器双节点部署，主节点故障时自动切换至备用节点；数据库主从切换时间 ≤5 分钟；定时全量备份（每周）+ 增量备份（每日），备份数据异地存储 |
| **系统运行故障** | 记录详细错误日志（时间、模块、错误码、堆栈信息）；关键操作失败后支持重试（如邮件/短信发送失败自动重试 3 次）；AI 审核服务不可用时自动降级为纯人工审核 |
| **用户操作不正确** | 表单实时校验，即时提示错误原因；关键操作（终止入职、合同推送等）二次确认；操作失误支持撤销（在限定时间内）；提供在线客服入口，随时获得帮助 |
| **外部系统故障** | 外部接口调用超时（30秒）自动熔断；失败请求进入重试队列（最多重试 3 次，间隔递增）；降级策略：电子签不可用时切换至线下签署模式 |
| **网络故障** | 移动端支持离线暂存，网络恢复后自动同步；弱网环境下基本功能可用（信息查看、表单填写） |
| **数据错误** | 数据异常时记录错误日志并通知管理员；支持数据修正流程（审批后修改）；定期进行数据一致性检查 |
